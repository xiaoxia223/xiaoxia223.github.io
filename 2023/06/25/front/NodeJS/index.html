Node模块化导入

简单使用

模块暴露数据（module.exports === exports）

导入模块注意事项

  • 导入自己模块的时候需要写相对路径
  • js、json文件导入的时候后缀可以省略
  • 导入其他类型文件时候会按照js文件处理
  • 如果导入的路径是个文件夹，那么首先会检测该文件夹下面的package.json文件中main属性对应的文件，如果main属性不存在或者package.json文件不存在，则会尝试导入文件夹下的index.js或者index.json文件，如果还是没找到，就会报错
  • 导入node.js内置模块时，直接require模块名字即可，无需路径

CommonJS规范

module.exports、exports以及require这些都是CommonJS模块化规范的内容，而Node.js是实现了CommonJS模块化规范，二者关系有点想JavaScript和ECMAScript

包管理工具

  • npm（Node Package Manager）（可以使用nrm工具切换镜像源）
  • yarn
  • cnpm（淘宝搭建的跟npm国内服务，有vpn几乎不考虑，或者更换Npm国内镜像源即可）

安装node后会内置npm包管理工具

在某个文件夹下面使用npm init 会进行包的初始化，以交互式的方式让我们生成package.json文件

初始化过程中一些注意事项

  1. Package name（包名）不能使用中文、大写字母，默认值是文件夹的名字，所以文件夹的名字也不能是中文
  2. version（版本号）要以x.x.x的形式定义，x必须是数字，默认值是1.0.0
  3. ISC证书与MIT证书功能上是相同的
  4. package.json可以手动创建或者修改
  5. 使用npm init -y 或者 npm init –yes 极速创建 package.json

npm

npm下载安装包

  • 下载安装之前首先需要初始化，也就是文件夹下面必须有package.json文件
  • 我们需要通过 npm i 和 npm install 来安装包

运行之后文件夹下会增加两个资源

  • node_modules文件夹 存放下载的包
  • package-lock.json 包的锁文件，用来锁定包版本

require导入npm包基本流程

  1. 在当前文件夹下的node_modules中寻找同名的文件夹
  2. 在上级目录中的node_modules中寻找同名文件夹、直至找到磁盘根目录

类型 命令 补充

npm i -save jquery		 -S 等效与save，-S是默认选项

npm i –save-dev less		 -D 等效与 –save-dev

全局安装

可以根据-g参数进行全局安装

全局安装完成之后就可以在命令行的任何位置运行nodemon参数

该命令的作用是自动重启node应用程序

  • 全局安装不受工作目录位置影响
  • 可以通过 npm root -g 来查看全局安装的位置
  • 不是所有的包都适合进行全局安装

Webpack打包工具

为什么需要打包工具？

开发时，我们会使用框架（React、vue），ES6模块化语法，Less、Sass等css预处理器等语法进行开发

这样的代码想要在浏览器运行必须经过编译成浏览器能够识别的js、css等语法才能运行

所以需要打包工具来帮助我们完成这些事。

除此之外，打包工具还能够压缩代码、做兼容性处理、提高代码性能等

有哪些打包工具

  • Grunt
  • Gulp
  • Parcel
  • Webpack
  • Rollup
